def balance_classes(X, y):
    values, counts = np.unique(y, return_counts=True)
    mode_class_count = max(counts)
    transforms = [rotate, noise, crop]  # all of these have a random component
    for class_ in values:
        new_imgs = []
        class_imgs = X[y == class_]
        diff_count = mode_class_count - counts[values == class_][0]
        for i in range(diff_count):
            original_img = class_imgs[random.randint(0, len(class_imgs)-1)]
            transform = random.choice(transforms)
            new_imgs.append(transform(original_img))
        if new_imgs:
            X = np.concatenate((X, np.array(new_imgs)))
            y = np.concatenate((y, [class_] * diff_count))
    return X, y
X_train, y_train = balance_classes(X_train, y_train)

Question 2¶

Describe how you set up the training, validation and testing data for your model. Optional: If you generated additional data, how did you generate the data? Why did you generate the data? What are the differences in the new dataset (with generated data) from the original dataset?

Answer:

I used the traditional 80/20 split (also suggested in class) for the training and validation set. The test set is already given. After that, I generated a lot of additional data for the training set (I do not want my validation set to contain fake data). These transformations were used to generate artificial data:

  • Blur: Gaussian blur with a 3x3 kernel.
  • Mirror: horizontal flip of the whole image.
  • Gray: image is converted to grayscale and then back to RGB, to keep the (32, 32, 3) shape.
  • Rotate: image is rotated r degrees, being r between [-15, 15].
  • Noise: a random value between [0, 15] is added to every color channel of every pixel.
  • Crop: a random 28x28 crop is taken from the original image, and then resize to 32x32.

Data augmentation helps increasing the data available for training, and can make the model generalize better. For example, random croppings can make the model more robust to changes in the position of the target inside the picture (translation invariance). I also used it to balance the classes, since data exploration showed great differences between the number of samples of each class.

The new training dataset consists on:

  • The original images remaining after the training/validation split.
  • For every class, new images are generated by applying random transforms (rotate, noise or crop) to random images of the class, until every class has the same number of training samples.
  • For every image, six new versions are added, one for each transformation.

The experiments performed with the new training dataset show that validation accuracy is improved by 1-2% with the LeNet network.

Note: another way of generating fake data is to randomly change the brightness and contrast of the pictures. It looks like a suitable technique in this case, since in the real world traffic signs are affected by lighting conditions, but I think the original dataset provides sufficient examples of this.

from tensorflow.contrib.layers import flatten

def LeNet(x, keep_prob):    
    # Hyperparameters
    mu = 0
    sigma = 0.1
    
    # SOLUTION: Layer 1: Convolutional. Input = 32x32x3. Output = 28x28x6.
    conv1_W = tf.Variable(tf.truncated_normal(shape=(5, 5, 3, 6), mean = mu, stddev = sigma))
    conv1_b = tf.Variable(tf.zeros(6))
    conv1   = tf.nn.conv2d(x, conv1_W, strides=[1, 1, 1, 1], padding='VALID') + conv1_b

    # SOLUTION: Activation.
    conv1 = tf.nn.relu(conv1)
    conv1 = tf.nn.dropout(conv1, keep_prob)

    # SOLUTION: Layer 1_5: Convolutional. Input = 28x28x6. Output = 20x20x6.
    conv1_5_W = tf.Variable(tf.truncated_normal(shape=(9, 9, 6, 6), mean = mu, stddev = sigma))
    conv1_5_b = tf.Variable(tf.zeros(6))
    conv1_5   = tf.nn.conv2d(conv1, conv1_5_W, strides=[1, 1, 1, 1], padding='VALID') + conv1_5_b

    # SOLUTION: Activation.
    conv1_5 = tf.nn.relu(conv1_5)
    conv1_5 = tf.nn.dropout(conv1_5, keep_prob)    
    
    # SOLUTION: Pooling. Input = 20x20x6. Output = 14x14x6.
    conv1_5 = tf.nn.avg_pool(conv1_5, ksize=[1, 7, 7, 1], strides=[1, 1, 1, 1], padding='VALID')

    # SOLUTION: Layer 2: Convolutional. Output = 10x10x16.
    conv2_W = tf.Variable(tf.truncated_normal(shape=(5, 5, 6, 16), mean = mu, stddev = sigma))
    conv2_b = tf.Variable(tf.zeros(16))
    conv2   = tf.nn.conv2d(conv1_5, conv2_W, strides=[1, 1, 1, 1], padding='VALID') + conv2_b
    
    # SOLUTION: Activation.
    conv2 = tf.nn.relu(conv2)
    conv2 = tf.nn.dropout(conv2, keep_prob)

    # SOLUTION: Pooling. Input = 10x10x16. Output = 5x5x16.
    conv2 = tf.nn.max_pool(conv2, ksize=[1, 2, 2, 1], strides=[1, 2, 2, 1], padding='VALID')

    # SOLUTION: Flatten. Input = 5x5x16. Output = 400.
    fc0   = flatten(conv2)
    
    # SOLUTION: Layer 3: Fully Connected. Input = 400. Output = 120.
    fc1_W = tf.Variable(tf.truncated_normal(shape=(400, 120), mean = mu, stddev = sigma))
    fc1_b = tf.Variable(tf.zeros(120))
    fc1   = tf.matmul(fc0, fc1_W) + fc1_b
    
    # SOLUTION: Activation.
    fc1    = tf.nn.relu(fc1)
    fc1 = tf.nn.dropout(fc1, keep_prob)

    # SOLUTION: Layer 4: Fully Connected. Input = 120. Output = 84.
    fc2_W  = tf.Variable(tf.truncated_normal(shape=(120, 84), mean = mu, stddev = sigma))
    fc2_b  = tf.Variable(tf.zeros(84))
    fc2    = tf.matmul(fc1, fc2_W) + fc2_b
    
    # SOLUTION: Activation.
    fc2    = tf.nn.relu(fc2)
    fc2 = tf.nn.dropout(fc2, keep_prob)

    # SOLUTION: Layer 5: Fully Connected. Input = 84. Output = 10.
    fc3_W  = tf.Variable(tf.truncated_normal(shape=(84, 43), mean = mu, stddev = sigma))
    fc3_b  = tf.Variable(tf.zeros(43))
    logits = tf.matmul(fc2, fc3_W) + fc3_b
    
    return logits

def evaluate(X_data, y_data):
    num_examples = len(X_data)
    total_accuracy = 0
    sess = tf.get_default_session()
    for offset in range(0, num_examples, BATCH_SIZE):
        batch_x, batch_y = X_data[offset:offset+BATCH_SIZE], y_data[offset:offset+BATCH_SIZE]
        accuracy = sess.run(accuracy_operation, feed_dict={x: batch_x, y: batch_y, keep_prob: 1.0})
        total_accuracy += (accuracy * len(batch_x))
    return total_accuracy / num_examples

Question 3¶

What does your final architecture look like? (Type of model, layers, sizes, connectivity, etc.) For reference on how to build a deep neural network using TensorFlow, see Deep Neural Network in TensorFlow from the classroom.

Answer:

The arquitecture of my model is basically the LeNet network we saw in the lab, that it is suggested as a starting point, with some additions:

  • I did not like the big downsample from 28x28x6 to 14x14x6, so I introduced a convolution layer (conv1_5) between the first one and the first pooling layer, that receives an input of 28x28x6 and outputs 20x20x6.
  • Thus, the first pooling layer has to be changed. I used ksize=[1, 7, 7, 1], strides=[1, 1, 1, 1], padding='VALID' to transform the input of 20x20x6 to an output of 14x14x6. Since 7x7 looks like a big pooling filter, I changed it to average pooling, instead of max pooling, hoping to capture less biased information. The rest of the network arquitecture stays the same.
  • I added dropout after every reLu.

Question 4¶

How did you train your model? (Type of optimizer, batch size, epochs, hyperparameters, etc.)

Answer:

  • Optimizer: I kept the AdamOptimizer from vanilla LeNet.
  • Batch size: after some trials with sizes of 64 and 256, it looks like the size of 128, suggested for LeNet, works better.
  • Epochs: after the introduction of dropout, it seems that the network needs more than the default 10 epochs to stabilize. I used 20, since different trials showed that is more than enough.
  • Learning rate: trials with learning rates of 0.01 (a total disaster) and of 0.0001 (no improvements) suggest that the default learning rate of 0.001 is quite good.
  • Dropout rate: the experiments performed suggest that a dropout rate of 0.9 after every reLu layer improves the validation accuracy.

Question 5¶

What approach did you take in coming up with a solution to this problem? It may have been a process of trial and error, in which case, outline the steps you took to get to the final solution and why you chose those steps. Perhaps your solution involved an already well known implementation or architecture. In this case, discuss why you think this is suitable for the current problem.

Answer:

I used a trial and error kind of process. First, I tried several preprocessing and data augmentation techniques using the base LeNet model for the trials, until I had the best dataset I could generate. After that, I tried to optimize some of the hyperparameters of the base LeNet model, but I could not find something better than the default values. Finally, I made some changes to the network structure, that seemed to make a minor improvement to the validation accuracy (from 97-98% to 98-99%). Outline of the results, with the best validation accuracy obtained in each case:

  • Preprocessing trials:
    • Base LeNet with RGB images: 0.955
    • Base LeNet with grayscale images: 0.963
    • Base LeNet with grayscale, min-max scaled images: 0.954
    • Base LeNet with grayscale, blurred images: 0.967 (decided to keep gray and blur transforms for data augmentation)
  • Data augmentation trials:
    • Base LeNet with RGB images, adding a blur version of every copy: 0.973
    • Also adding a mirrored version of each image: 0.964
    • Also adding a grayscale version of each image: 0.967
    • Also adding a randomly rotated version of each: 0.973
    • Also adding a noisy version of each: 0.980
    • Also adding a cropped version of each: 0.983
    • Adding more versions transformed with rotate, noise and crop, to balance the classes: 0.983
  • Hyperparameter optimization trials:
    • Trying with 20 epochs: 0.982
    • Trying a batch size of 256: 0.980
    • Trying a batch size of 64: 0.983
    • Trying a learning rate of 0.0001 and 20 epochs: 0.977
    • Trying a learning rate of 0.01 and 10 epochs: 0.056
  • Changes to the structure:
    • Dropout of 0.5 after every reLu, 20 epochs: 0.845
    • Adding a 28x28x6 to 20x20x6 conv layer and adapting the first pooling layer, no dropout, 10 epochs: 0.986
    • Changing the first pooling layer (max pooling) to average pooling: 0.986 (although better average accuracy than in the previous trial)
    • Adding a dropout rate of 0.9: 0.990
    • Trying 20 epochs with the same dropout: 0.992
    • Trying a dropout rate of 0.8, same number of epochs: 0.986

The test accuracy is around 94% with the final model, which indicates that there might be some overfitting.

Question 8¶

Use the model's softmax probabilities to visualize the certainty of its predictions, tf.nn.top_k could prove helpful here. Which predictions is the model certain of? Uncertain? If the model was incorrect in its initial prediction, does the correct prediction appear in the top k? (k should be 5 at most)

tf.nn.top_k will return the values and indices (class ids) of the top k predictions. So if k=3, for each sign, it'll return the 3 largest probabilities (out of a possible 43) and the correspoding class ids.
